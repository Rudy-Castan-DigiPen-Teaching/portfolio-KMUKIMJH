Project Procedural Geometric Modeling

3D Graphics

Project Overview

The Procedural Meshes project was a part of my CS250 Computer Graphics course at DigiPen. The goal of this assignment was to mathematically generate and render various 3D primitive meshes dynamically at runtime for both Windows and Web platforms.

Tasks Completed

  • Implemented procedural generation algorithms using parametric equations for a Plane, Cube, Sphere, Torus, Cylinder, Cone, and a custom Trefoil Knot.
  • Constructed index buffers and managed mesh topologies to support both solid triangle rendering and wireframe line rendering.
  • Integrated the generated geometries into the application using a factory pattern and connected them to an ImGui interface for real-time resolution control.
  • Resolved strict cross-platform compilation errors for WebGL (Emscripten) by enforcing rigorous C++ type safety and preventing implicit conversions.

Reflection

During development, I encountered a critical issue where the WebGL build would fail due to Emscripten's strict compiler warnings regarding implicit type conversions and floating-point promotions. By meticulously refactoring the codebase to enforce strict type safety, I was able to achieve a flawless cross-platform build. Additionally, integrating the bonus Trefoil Knot required careful spatial transformations to correctly align its scale and position with the other primitives. This project significantly enhanced my understanding of 3D parametric equations, mesh topology, and writing robust, cross-platform C++ code.
